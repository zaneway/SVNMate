# Limitations

本文档明确说明 SVNMate 当前版本的能力边界，帮助使用者和贡献者建立正确预期。

## 1. 产品边界

SVNMate 当前是一个“工作副本操作工具”，而不是完整的 SVN 管理平台。

当前重点支持：

- 打开工作副本
- checkout
- 状态浏览
- diff
- 批量 add / commit
- update / cleanup / resolve

当前未覆盖：

- branch / tag 管理
- merge assistant
- 图形化历史浏览
- 仓库浏览器

## 2. 认证与凭据

当前版本没有独立的认证管理 UI，也没有 Keychain 凭据管理界面。

这意味着：

- 认证行为主要依赖本地 `svn` 命令的现有能力
- 某些认证失败场景会以 SVN 原始错误形式暴露
- 对复杂证书、代理、交互式认证的可用性仍有限

## 3. 本地化边界

当前支持语言：

- `zh-Hans`
- `en`

未支持的系统语言会回退到简体中文。当前不会对以下内容做全文翻译：

- SVN 服务器返回的原始 stderr
- 所有极少触发的内部错误文本

## 4. 差异展示边界

当前 diff 视图是文本型 diff，适合：

- 代码文件
- 文本文件

当前不适合：

- 二进制文件
- 图片
- Word / Excel / PowerPoint 等 Office 文档的可视化对比

## 5. Tree Conflict 处理边界

当前版本支持：

- 识别 tree conflict
- 展示 tree conflict detail
- 执行 `svn resolve --accept working`

当前不支持：

- 多策略冲突解决向导
- 图形化合并
- 冲突内容可视化编辑

## 6. 构建与分发边界

仓库内置的 DMG 打包链路当前定位为：

- 本地开发验证
- 内部测试分发

当前不包含：

- Developer ID 正式签名
- notarization
- stapling
- 外部正式发布流程

因此，未签名或未公证的包在目标机器上可能触发 Gatekeeper 提示。

## 7. 运行环境边界

SVNMate 明确依赖本地 `svn` 二进制，而不是内嵌 SVN 库。

这带来的约束是：

- 目标环境必须有可用的 `svn`
- `svn` 的路径和版本会影响实际行为
- 不同企业内网环境中的证书、代理、凭据策略会影响体验

## 8. 当前适用场景

当前版本更适合：

- 已有 SVN 基础设施的 macOS 团队
- 需要一个轻量桌面工作副本工具的研发和文档团队
- 内部测试和持续迭代中的桌面客户端项目

当前不适合直接视为：

- 面向最终大众用户的正式商业成品
- 全量替代成熟 SVN 企业客户端的完整方案
