# SVNMate 使用手册

## 1. 适用范围

本手册面向使用 SVNMate 管理本地 SVN 工作副本的研发人员、测试人员和技术支持人员。

当前版本适用于以下典型操作：

- 打开已有工作副本
- 从仓库地址检出到本地目录
- 查看工作副本状态
- 查看文件差异
- 选择部分文件提交
- 执行更新、清理、冲突标记解决

## 2. 使用前准备

启动 SVNMate 之前，请确认：

- macOS 版本为 13.0 及以上
- 本机已安装 `svn`
- 当前用户对工作副本目录具有读写权限

可在终端执行以下命令验证：

```bash
svn --version
```

如果系统提示找不到命令，请先参考安装部署手册安装 SVN。

## 3. 首次进入应用

首次打开应用时，主界面右侧会显示欢迎页，左侧仓库列表为空。

可通过两种方式开始使用：

- `New Checkout`：从远程仓库地址检出到本地
- `Open Repository`：打开一个已经存在的本地 SVN 工作副本

## 4. 打开已有仓库

### 4.1 操作步骤

1. 点击左侧或顶部菜单中的 `Open Repository...`
2. 在文件选择框中选择一个本地工作副本目录
3. 应用会自动执行 `svn info`
4. 验证通过后，该仓库会加入左侧仓库列表

### 4.2 成功结果

- 左侧出现仓库名称与路径
- 右侧进入仓库详情页
- 系统自动加载该工作副本的状态树

### 4.3 常见失败原因

- 选择的目录不是 SVN 工作副本
- 本机未安装 `svn`
- `svn` 路径无法被客户端识别

## 5. 检出远程仓库

### 5.1 操作步骤

1. 点击 `New Checkout`
2. 输入仓库 URL
3. 选择本地目录
4. 点击 `Checkout`

### 5.2 使用建议

- 建议选择一个空目录或新建目录作为检出路径
- 如果目录中已有其他文件，建议先自行确认不会发生内容覆盖或混淆

### 5.3 检出完成后

检出成功后，客户端会自动尝试将该目录作为本地仓库加入列表并切换到详情页。

## 6. 浏览文件状态

仓库详情页左侧为文件状态树，展示当前工作副本中发生变化的目录和文件。

### 6.1 状态说明

| 状态 | 含义 |
|------|------|
| Modified | 文件内容已修改 |
| Added | 文件已加入版本控制 |
| Deleted | 文件已删除 |
| Unversioned | 文件未纳入版本控制 |
| Conflict | 文件存在冲突 |
| Ignored | 文件被忽略 |
| Missing | 版本控制中存在，但本地缺失 |
| Replaced | 文件被替换 |
| External | 外部引用 |

### 6.2 目录展示逻辑

如果实际变更文件位于多级目录下，而父目录本身没有直接出现在 SVN 状态输出中，客户端会自动补齐父目录，保证文件树可以正常浏览。

## 7. 查看文件差异

### 7.1 操作步骤

1. 在文件树中点击目标文件
2. 右侧详情区点击 `View Diff`

### 7.2 结果说明

- 下方会显示该文件的 diff 文本
- diff 以等宽字体展示
- 当前版本为文本 diff，不提供图形化对比视图

## 8. 选择性提交

### 8.1 文件勾选规则

在文件树中，只有可提交文件才会显示复选框。当前默认允许勾选的状态包括：

- Modified
- Added
- Deleted
- Replaced

以下状态默认不参与勾选提交：

- Unversioned
- Ignored
- Conflict
- Missing
- External

### 8.2 提交流程

1. 在文件树中勾选需要提交的文件
2. 点击工具栏中的 `Commit`
3. 在弹窗中填写提交说明
4. 确认提交

### 8.3 不勾选文件时的行为

如果提交前没有勾选任何文件，客户端会按仓库范围执行提交，即提交当前工作副本中所有待提交的版本化变更。

### 8.4 使用建议

- 建议每次提交前先查看 diff
- 建议一次提交只包含单一目的变更，避免混合多个需求或修复
- 如果工作副本存在冲突，请先处理冲突，再执行提交

## 9. 更新工作副本

### 9.1 操作步骤

1. 打开目标仓库
2. 点击工具栏中的 `Update`

### 9.2 作用

- 将工作副本更新到服务器最新状态
- 更新完成后客户端会自动重新加载文件状态树

### 9.3 注意事项

- 如果本地有未提交修改，更新过程中可能产生冲突
- 建议在重要修改前先执行一次 update

## 10. Cleanup

### 10.1 适用场景

如果工作副本因中断、锁定或异常状态出现问题，可尝试执行 `Cleanup`。

### 10.2 操作步骤

1. 打开目标仓库
2. 点击工具栏中的 `Cleanup`

执行完成后，客户端会自动刷新状态。

## 11. 冲突处理

### 11.1 当前支持

如果文件状态为 `Conflict`，在右侧详情区会显示 `Resolve` 按钮。

### 11.2 当前行为

当前版本执行的是：

```bash
svn resolve --accept working <file>
```

即保留 working copy 版本并标记冲突已解决。

### 11.3 使用建议

执行 `Resolve` 前，建议先自行检查冲突文件内容，确认工作副本内容符合预期。

## 12. 仓库管理

### 12.1 删除仓库记录

在左侧仓库列表中，可对仓库执行 `Remove`。

### 12.2 行为说明

`Remove` 只会移除客户端中的记录，不会删除本地工作副本目录。

## 13. 错误提示与故障排查

### 13.1 找不到 svn

表现：

- 打开仓库、检出、刷新时提示无法找到 `svn`

处理：

- 确认本机已安装 SVN
- 确认安装路径为 `/usr/bin/svn`、`/opt/homebrew/bin/svn` 或 `/usr/local/bin/svn`

### 13.2 目录不是工作副本

表现：

- 打开目录时报错

处理：

- 在终端进入该目录执行 `svn info`
- 如果命令失败，说明该目录不是合法工作副本

### 13.3 提交失败

常见原因：

- 提交说明为空
- 工作副本存在冲突
- 服务器认证失败
- 目标文件状态不允许提交

### 13.4 更新失败

常见原因：

- 网络不可达
- 认证失败
- 本地存在冲突或锁定

## 14. 当前版本限制

当前版本暂未提供以下 UI 能力：

- 历史日志浏览
- 仓库 revision 信息面板
- 用户名密码或证书配置页
- 细粒度冲突解决策略选择
- 图形化 diff 视图
- branch/tag 操作

## 15. 建议使用规范

- 每次开始工作前先 update
- 提交前先 diff
- 尽量按变更主题勾选文件，避免大杂烩提交
- 对于生成文件、临时文件和日志文件，优先在仓库层面做好 ignore 策略
